# Resources

Navigation: [[index]] | [[projects/_index]] | [[areas/_index]] | [[archives/_index]]

Reference material with no deadline and no owner responsibility — things you want to be able to find, not things you're actively driving. Under PARA mode, new content routes here:

- `wiki/resources/incoming/` — new ingested sources land here for topical sorting
- `wiki/resources/people/` — entity pages for people
- `wiki/resources/concepts/` — concept/framework pages
- `wiki/resources/<topic>/` — create topic subfolders as they emerge (e.g. `resources/tools/`, `resources/health/`)

## Pre-existing reference content (generic mode, not yet migrated)

This vault ran in **generic** mode before switching to PARA on 2026-07-08. The following folders hold pre-PARA content and were intentionally left in place (no auto-migration — see [[../../docs/methodology-modes-guide.md|methodology modes guide]]):

- [[../concepts/_index|wiki/concepts/]] — concept pages from the claude-obsidian plugin's own development history
- [[../entities/_index|wiki/entities/]] — people, repos, and tools referenced during that history
- [[../sources/_index|wiki/sources/]] — ingested source summaries
- `wiki/comparisons/` — side-by-side analyses
- `wiki/questions/` — filed answers to past queries

Move any of these into `resources/<topic>/` (or into a project/area) manually when you triage them — see PARA migration note in [[../overview|overview]].
